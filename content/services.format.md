# Services content format

Use `services.md` for service cards. Each service is a level-3 heading followed by one or more description paragraphs.

```md
### <emoji> <Service name>
<Short value proposition>.<br>
<Additional scope, audience, or delivery detail>.
```

Rules:
- Keep one blank line between service blocks.
- Use `<br>` only when the current layout needs a deliberate line break inside the same service description.
- Do not add unordered lists unless the renderer is updated to support them.
